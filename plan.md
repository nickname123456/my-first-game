# План разработки игры

## Этап 1 — Скелет игры

**Шаг 1 — pygame с нуля**
- Установка pygame, структура проекта
- Игровой цикл: окно, FPS, выход
- Спрайт игрока + движение WASD

**Шаг 2 — Офис**
- Статичный фон (прямоугольники = комнаты, пока без картинок)
- Коллизии со стенами
- Зоны: рабочая зона, кухня, переговорка

**Шаг 3 — Сотрудники**
- 3 сотрудника-спрайта стоят на своих местах
- У каждого: имя, навык (dev / design / qa), состояние (idle / working / problem)
- Визуальный индикатор состояния над головой

**Шаг 4 — Задачи**
- Класс `Task`: название, навык, сложность, дедлайн
- Генератор задач (пул из ~10 заготовленных)
- Новая задача появляется каждые N секунд

**Шаг 5 — Канбан-доска**
- Доска в углу офиса как объект-зона
- Игрок подходит -> нажимает E -> открывается UI-оверлей со списком задач
- Задачи отображаются с параметрами, можно выбрать мышью

**Шаг 6 — Назначение задач**
- Выбрал задачу -> выбираешь сотрудника (список или клик по нему)
- Задача "вешается" на сотрудника, он переходит в состояние `working`
- Прогресс-бар над головой

**Шаг 7 — Буфер / доделка этапа**
- Исправить баги первых 6 шагов
- Убедиться, что базовый цикл "взял задачу -> назначил -> прогресс идёт" работает

## Этап 2 — Логика и финал

**Шаг 8 — Статы**
- Три полоски: Бюджет, Качество, Мораль
- Правила влияния: задача выполнена вовремя -> +качество, просрочена -> −бюджет, неверный навык -> −качество, не помог сотруднику -> -мораль

**Шаг 9 — События**
- 3 типа событий: "пожар в коде", "выгорание", "конфликт"
- Событие появляется случайно, у сотрудника иконка тревоги
- Игрок должен подбежать и нажать E в течение таймаута

**Шаг 10 — Решения по событиям**
- UI-меню с выбором: "Помочь самому" / "Позвать коллегу" / "Игнорировать"
- Каждый выбор имеет последствие для статов

**Шаг 11 — Победа и поражение**
- Условия конца игры: Мораль = 0 -> поражение, Бюджет = 0 -> поражение
- Финальный проект "релизится" через 5 минут -> победа если все статы > 0
- Экраны Game Over и Victory

**Шаг 12 — Ассеты**
- Заменить прямоугольники на простые пиксельные спрайты (OpenGameArt.org)
- Или оставить минималистичный стиль — тоже работает

**Шаг 13 — Звук и фидбек**
- Простые звуки (pygame.mixer): уведомление, назначение задачи, событие
- Всплывающие текстовые уведомления на экране

**Шаг 14 — Тест и полировка**
- Полный прогон игры от начала до конца
- Баланс: скорость задач, сложность событий
- Финальный тест
