# my-first-game
Юзер играешь за проджект-менеджера в маленьком IT-стартапе. Вид сверху, пиксельная графика, простой офис с зонами: рабочая зона, кухня, переговорка.

**Как это работает:**
1. **Доска задач (Канбан):** В углу офиса висит доска. Туда прилетают новые задачи с параметрами: срочность, сложность, нужный навык, дедлайн.
2. **Планирование:** Ты подходишь к доске, берёшь задачу и назначаешь её конкретному сотруднику — с учётом его навыков и текущей загрузки.
3. **Наблюдение:** После назначения ты ходишь по офису и смотришь, как команда работает. Сотрудники сами выполняют задачи — ты не микроменеджишь каждый клик, а контролируешь процесс.
4. **Проблемы:** Иногда что-то идёт не так: у разработчика «пожар» в коде, дизайнер выгорел, в переговорке назревает конфликт, упал сервер. Ты должен быстро подбежать, оценить ситуацию и принять решение: помочь самому, позвать другого сотрудника, выделить ресурс или проигнорировать (на свой страх и риск).
5. **Цель:** Успешно довести проект до релиза, не уронив бюджет, качество продукта и мораль команды.
